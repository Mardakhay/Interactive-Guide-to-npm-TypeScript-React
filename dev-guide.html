01

What is a Package Manager?

💡The Core Problem

Imagine you're building an app that needs to format dates. You have two choices:

  • Write it yourself — takes days, full of edge cases (timezones? leap years? 😰)
  • Use someone else's library — working in seconds, battle-tested by thousands
A package (or library) is a reusable chunk of code someone published so you don't have to write it yourself. A package manager is the tool that downloads, installs, and manages those packages for you.
📦Before Package Managers (the dark ages)
1
Find the library website

Search Google, maybe it's on GitHub, maybe a random server...

2
Download a zip file

Pray that the download link still works.

3
Manually copy files into your project

Don't forget that this library depends on another library... repeat.

4
Check for updates every month manually 😭
npm solves ALL of this with a single command. It downloads, tracks versions, resolves dependencies of dependencies, and verifies integrity automatically.

03

Using npm — Hands On

⚙Step 1: Initialize a Project

Every project starts with package.json — the project's "manifest" file. Create it with:

terminal
$ npm init -y
Wrote to /my-project/package.json:
{
  "name": "my-project",
  "version": "1.0.0",
  "scripts": {},
  "dependencies": {}
}
✓ Done!

The -y flag means "yes to everything" — skips the Q&A wizard.

⬇Step 2: Install a Package
terminal
$ npm install date-fns
added 1 package in 0.8s
✓ package installed!

$ npm install eslint --save-dev
added 212 packages in 3.2s
✓ dev dependency installed!
dependency
npm install <name>
Needed when your app runs. Like date-fns for formatting dates.
devDependency
npm install <name> --save-dev
Only needed during development. Like eslint (code linter).
📁What npm creates in your project
📄 package.json — Lists your dependencies & their required versions
🔒 package-lock.json — The lock file: exact versions of every package installed
📁 node_modules/ — All downloaded packages. Never commit this!
⚠️ Always commit your lock file (package-lock.json) to git. This ensures everyone on your team installs the exact same package versions — preventing "works on my machine" bugs.
▶npm Scripts — Your Task Runner

You can define shortcut commands in package.json under "scripts":

package.json
{ "scripts": { "start": "node index.js", "dev": "parcel index.html", "build": "webpack --mode production", "test": "jest" } }
terminal
$ npm run dev # runs "parcel index.html"
$ npm run build # runs "webpack --mode production"
$ npm start # special shortcut (no "run" needed)

● Chapter 2

TypeScript &
Static Typing

JavaScript is flexible — sometimes too flexible. TypeScript adds types so your editor catches bugs before they reach users.

🔷
01

The Problem TypeScript Solves

💥JavaScript's Hidden Traps

JavaScript is dynamically typed — variables can hold any type, and types can change at runtime. This causes sneaky bugs:

JavaScript — the bug you won't see coming
function generateEmail(user) { return `${user.name}@company.com`; } // Developer calls it with wrong field name 👇 generateEmail({ fullName: 'Petro Nikolsky' }); // Returns: "undefined@company.com" 😱 // No error! Bug silently goes to production!
TypeScript — caught before it runs
function generateEmail(user: { name: string }) { return `${user.name}@company.com`; } generateEmail({ fullName: 'Petro Nikolsky' }); // ❌ ERROR at compile time: // Property 'fullName' does not exist on type '{ name: string }' // You fix it BEFORE it ships! ✅
🕐 The earlier you catch a bug, the cheaper it is to fix. TypeScript catches type errors at write time — right in your editor, before you even run the code.
02

TypeScript Syntax — The Essentials

📝Typing Variables

Add a type after a colon : type

TypeScript
// Basic types let age: number = 25; let name: string = 'Alice'; let isAdmin: boolean = false; // Arrays let scores: number[] = [95, 87, 92]; let tags: string[] = ['frontend', 'react']; // TypeScript can also INFER types — you don't always need to write them! let city = 'Berlin'; // TypeScript knows this is a string city = 42; // ❌ ERROR: Type 'number' is not assignable to type 'string'
⚙Typing Functions
TypeScript
// Parameter types + return type function add(a: number, b: number): number { return a + b; } // Arrow function with types const greet = (name: string): string => { return `Hello, ${name}!`; } // void = function returns nothing function log(message: string): void { console.log(message); }
🏗Interfaces — Defining Object Shapes

An interface is like a contract — it describes what properties an object must have.

TypeScript
interface User { id: number; name: string; email: string; age?: number; // ? means optional } function displayUser(user: User): void { console.log(`${user.name} — ${user.email}`); } // ✅ Valid displayUser({ id: 1, name: 'Alice', email: 'alice@dev.com' }); // ❌ Error: missing 'email' displayUser({ id: 2, name: 'Bob' });
🔀Union Types — "This OR That"
TypeScript
// id can be a string OR a number (common with APIs!) let id: string | number; id = 42; // ✅ id = 'abc123'; // ✅ id = true; // ❌ Error! // Status can only be one of these exact strings type Status = 'loading' | 'success' | 'error'; let status: Status = 'loading'; // ✅ status = 'pending'; // ❌ Not in the union!

● Chapter 3

React & UI
Components

React is a JavaScript library for building user interfaces. It lets you build complex UIs from small, reusable pieces called components.

⚛
01

Why React?

🤔The Problem React Solves

Imagine a "likes" button. Without React, you'd write something like this:

Vanilla JS — messy DOM manipulation
let likes = 0; const btn = document.getElementById('like-btn'); const counter = document.getElementById('like-count'); btn.addEventListener('click', () => { likes++; counter.textContent = likes; // manually update DOM if (likes > 0) btn.classList.add('liked'); // manually update style // now imagine 100 features like this... 😫 });

With React, you just describe what the UI should look like based on state, and React handles the DOM updates for you:

React — declarative & clean
function LikeButton() { const [likes, setLikes] = useState(0); return ( <button onClick={() => setLikes(likes + 1)}> ❤ {likes} Likes </button> ); } // React automatically re-renders when likes changes ✨
02

The Core Concepts

🧩Components — Building Blocks

Everything in React is a component — a JavaScript function that returns UI (written in JSX).

React Components
// A component is just a function that returns JSX function Greeting() { return <h1>Hello, World!</h1>; } // Components can receive data via "props" function UserCard({ name, role }) { return ( <div> <h2>{name}</h2> <p>{role}</p> </div> ); } // Use components like HTML tags function App() { return ( <div> <UserCard name="Alice" role="Developer" /> <UserCard name="Bob" role="Designer" /> </div> ); }
JSX looks like HTML but it's actually JavaScript. {curly braces} let you escape into JavaScript anywhere inside JSX.
🔄State — Data that Changes

State is data that belongs to a component and can change over time. When state changes, React automatically re-renders the component.

⚛ Live React-style Counter Demo
0
State: 0 → React re-renders automatically
useState Hook
import { useState } from 'react'; function Counter() { // useState(0) → initial value is 0 // count → current value (read-only!) // setCount → function to update it const [count, setCount] = useState(0); return ( <div> <p>Count: {count}</p> <button onClick={() => setCount(count + 1)}>+</button> <button onClick={() => setCount(count - 1)}>-</button> </div> ); }
⚠️ Never modify state directly! Always use the setter function.
count = count + 1 ❌ — React won't know it changed
setCount(count + 1) ✅ — React re-renders correctly
📩Props — Passing Data Down

Props are how components communicate. A parent component passes data to children via props (like HTML attributes).

Props Example
// Child component — receives props function Badge({ text, color = 'blue' }) { // default value! return ( <span style={{ background: color }}> {text} </span> ); } // Parent component — passes props function App() { return ( <div> <Badge text="New" color="green" /> <Badge text="Hot" color="red" /> <Badge text="Sale" /> {/* uses default color='blue' */} </div> ); }
🔁Rendering Lists

To render a list of items, use JavaScript's .map() method inside JSX:

Lists in React
const fruits = ['🍎 Apple', '🍌 Banana', '🍊 Orange']; function FruitList() { return ( <ul> {fruits.map((fruit, index) => ( <li key={index}>{fruit}</li> // key= is required! React uses it to track items ))} </ul> ); }
🌐useEffect — Side Effects

Want to fetch data when a component loads? Or run code when something changes? Use useEffect:

useEffect Hook
import { useState, useEffect } from 'react'; function UserProfile({ userId }) { const [user, setUser] = useState(null); useEffect(() => { // Runs after the component renders fetch(`/api/users/${userId}`) .then(res => res.json()) .then(data => setUser(data)); }, [userId]); // ← dependency array: re-run when userId changes if (!user) return <p>Loading...</p>; return <h1>{user.name}</h1>; }
The dependency array [] controls when the effect runs:
• [] empty — run once when component mounts (like page load)
• [userId] — re-run whenever userId changes
• no array — run after every render (usually too much!)

React + TypeScript Together

💎Typing Props with TypeScript

import { useState } from 'react'; // Define the shape of props interface ButtonProps { label: string; onClick: () => void; disabled?: boolean; } // Type the props parameter function Button({ label, onClick, disabled = false }: ButtonProps) { return ( <button onClick={onClick} disabled={disabled}> {label} </button> ); } // TypeScript + useState function App() { const [count, setCount] = useState<number>(0); return ( <Button label="Click me" onClick={() => setCount(count + 1)} /> ); }
